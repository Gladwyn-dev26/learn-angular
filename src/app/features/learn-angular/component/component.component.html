Angular Component

A Component is the fundamental building block of an Angular application. It controls a part of the user interface and contains template, styles, and business logic.

1. What is a Component?

A component defines a portion of the UI and controls how the view behaves. Angular applications are built using a tree of components.

AppComponent
├ HeaderComponent
├ SidebarComponent
└ DashboardComponent

2. Why We Use Components

Reusability Maintainability Scalability Separation of Concerns

3. How Components Work

User Action arrow_forward Component Logic arrow_forward State Change arrow_forward Change Detection arrow_forward DOM Update

4. Component Syntax


@Component({
 selector: 'app-user',
 templateUrl: './user.component.html',
 styleUrls: ['./user.component.scss']
})
export class UserComponent {}

5. Standalone Components (Angular 19)

Standalone components remove the need for NgModules and simplify Angular architecture. 


@Component({
 selector: 'app-user',
 standalone: true,
 templateUrl: './user.component.html'
})
export class UserComponent {}
